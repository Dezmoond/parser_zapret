# Система парсеров для сбора данных

Система из 6 независимых парсеров для сбора данных с различных ресурсов и сохранения их в формате CSV.

## Структура проекта

```
.
├── config.json          # Единый файл конфигурации
├── main.py             # Главный файл для запуска всех парсеров
├── requirements.txt    # Зависимости проекта
├── parsers/           # Директория с парсерами
│   ├── __init__.py
│   ├── base_parser.py # Базовый класс для всех парсеров
│   ├── parser1.py     # Парсер 1 - HTML парсер
│   ├── parser2.py     # Парсер 2 - JSON API парсер
│   ├── parser3.py     # Парсер 3 - XML парсер
│   ├── parser4.py     # Парсер 4 - CSV парсер
│   ├── parser5.py     # Парсер 5 - Текстовый парсер с regex
│   └── parser6.py     # Парсер 6 - Универсальный парсер
├── utils/             # Утилиты
│   ├── __init__.py
│   └── csv_writer.py  # Утилита для записи в CSV
└── output/            # Директория для выходных CSV файлов (создается автоматически)
```

## Установка

1. Создайте виртуальное окружение (если еще не создано):
```bash
python -m venv venv
```

2. Активируйте виртуальное окружение:
```bash
# Windows
venv\Scripts\activate

# Linux/Mac
source venv/bin/activate
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. **Для Parser1 (парсер Росфинмониторинга) требуется ChromeDriver:**
   - Установите Google Chrome браузер
   - ChromeDriver будет автоматически установлен через `webdriver-manager` (уже включен в requirements.txt)
   - Альтернативно: скачайте ChromeDriver вручную с https://chromedriver.chromium.org/ и добавьте в PATH

## Конфигурация

Отредактируйте файл `config.json` для настройки парсеров:

- `urls` - список URL для каждого парсера
- `output_file` - путь к выходному CSV файлу
- `enabled` - включен/выключен парсер
- `settings` - глобальные настройки (timeout, retry_count, delay_between_requests, user_agent)

Пример конфигурации:
```json
{
  "parsers": {
    "parser1": {
      "name": "Парсер 1",
      "urls": ["https://example.com/data1"],
      "output_file": "output/parser1_data.csv",
      "enabled": true
    }
  },
  "settings": {
    "timeout": 30,
    "retry_count": 3,
    "delay_between_requests": 1
  }
}
```

## Использование

Запустите все парсеры:
```bash
python main.py
```

Каждый парсер будет:
1. Загружать данные с указанных URL
2. Парсить данные в соответствии со своей логикой
3. Сохранять результаты в CSV файл

## Типы парсеров

- **Parser1**: Парсер каталога террористов и экстремистов Росфинмониторинга
  - Использует Selenium для работы с динамическим контентом
  - Парсит два раздела: "Организации" и "Физические лица"
  - Сохраняет данные в два отдельных CSV файла в папку "Перечень террористов и экстремистов РОСФИНМОНИТОРИНГ"
  - Формат организаций: номер, наименование
  - Формат физических лиц: номер, ФИО, дата рождения, место рождения
- **Parser2**: Парсит JSON API ответы
- **Parser3**: Парсит XML данные
- **Parser4**: Парсит CSV данные
- **Parser5**: Парсит текстовые данные с помощью регулярных выражений
- **Parser6**: Универсальный парсер с автоматическим определением типа контента

## Логирование

Все логи сохраняются в файл `parsers.log` и выводятся в консоль.

## Расширение

Для создания нового парсера:
1. Создайте новый файл в директории `parsers/` (например, `parser7.py`)
2. Наследуйтесь от класса `BaseParser`
3. Реализуйте метод `parse(response)`
4. Добавьте парсер в `PARSER_CLASSES` в `main.py`
5. Добавьте конфигурацию в `config.json`
